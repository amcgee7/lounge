Trouble with invalid ssl.

These instructions were written with Ubuntu 16.04 in mind.  You must have the package openssh installed.

One solution to this problem is that the root CA is not install ed on your system and will need to be installed.

  1. Get this root CA for letsencrypt X3.  If the link doesn't work try here or search for letsencrypt root certificate. Make sure you get the pem file.
  2. This file or a link to the file belongs in /etc/ssl/certs/ folder.  Note on other distrabutions the location may change.
  3. Once done you will need to run sudo c_rehash to have the CA applied.  
  4. Testing is easy run openssl s_client -connect irc.jlrngi.com:6697 and this sould tell you the cert is now verified.
